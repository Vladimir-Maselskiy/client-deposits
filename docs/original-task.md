Тестове завдання: Модуль «Депозити клієнта» (Middle Fullstack JS) 🎯 Огляд
задачі Необхідно розробити веб-додаток для управління банківськими депозитами.
Система має включати перегляд існуючих вкладів та процес відкриття нового
депозиту через багатокрокову форму з імітацією реальної банківської транзакції.

🛠 Технологічний стек ● Framework: Next.js (App Router). ● Language: TypeScript.
● State Management: React Query та Zustand. ● Forms: React-Hook-Form + Zod
(валідація). ● UI: Material UI (MUI). ● Database: будь яка БД з якою ви вмієте
працювати. ● Infrastructure: Docker & Docker Compose. 📋 Основні функціональні
вимоги

1. Робота з базою даних (Data Layer) В базі даних мають бути реалізовані
   наступні сутності: ● Users: (id, ПІБ, адреса, список карток* із назвою,
   балансом та валютою). * - Реалізацію структури User+Cards залишаємо на ваш
   розсуд ● DepositPrograms: доступні банківські продукти (назва, ставка,
   термін, валюта). ● Contracts: оформлені договори користувачів. Примітка:
   Оскільки авторизація є додатковою умовою, за замовчуванням додаток має
   працювати з першим користувачем, знайденим у БД.

2. Сторінка «Мої вклади» ● Empty State: Якщо у користувача ще немає відкритих
   договорів, відобразити повідомлення: «У вас ще немає відкритих вкладів, але
   це легко виправити» та кнопку переходу до відкриття. ● Список: Якщо вклади є
   — відобразити їх перелік (назва програми, сума, дата створення).
3. Процес відкриття депозиту (Multi-step flow) Стан форми між кроками має
   зберігатися в Zustand. ● Крок 1: Вибір програми. Список доступних програм,
   завантажений із БД. ● Крок 2: Параметри. Поля: «Сума», «Назва» (опційно),
   «Спосіб внесення» (Готівка/Картка). Якщо обрано «Картка» — Select із картками
   користувача. ● Крок 3: Підтвердження. Екран перевірки всіх введених даних. ●
   Крок 4: Договір. Текст договору з динамічними даними клієнта та обраними
   параметрами вкладу + чекбокс згоди. Текст договору надано в додатку нижче.

4. Обробка транзакції (Async Flow) ● Після натискання «Оформити» викликається
   API для запису в БД. ● Емуляція затримки: Сервер має відповідати рівно 10
   секунд. ● Весь час очікування на фронтенді відображається спінер, інтерфейс
   блокується. ● Результат: При успіху — повідомлення з деталями вкладу та
   редирект на список «Мої вклади». При помилці — вивід повідомлення про
   помилку. 📦 Docker та розгортання ● Проект має запускатися командою:
   docker-compose up. ● Seeding: При старті база повинна автоматично
   наповнюватися тестовими даними (користувач, його картки, мінімум 3 депозитні
   програми). 🌟 Додаткові умови (не обов'язкові, за бажанням) ● Авторизація:
   Реалізація сторінки Login/Password (NextAuth або власний механізм на
   JWT/Cookies). ● Мобільна адаптація: Адаптивний інтерфейс для коректного
   відображення на смартфонах. ● Swagger: Документація API ендпоінтів. 🔍
   Критерії оцінки Архітектура: Розподіл бізнес-логіки, робота з шаром даних.
   Стейт-менеджмент: Коректне поєднання React Query та Zustand. Валідація:
   Використання Zod для складних та умовних перевірок. DevOps: Робота з Docker,
   міграціями та сідінгом бази. Clean Code: Типізація, структура проекту та
   якість коду. Додатки Текст договору ЗАЯВА-АНКЕТА на відкриття банківського
   вкладу Дані клієнта ПІБ: [ClientFullName] Адреса реєстрації: [ClientAddress]
   Я, [ClientFullName], прошу відкрити мені депозитний рахунок згідно з обраними
   умовами та підтверджую достовірність вказаних даних. Умови вкладу Назва
   програми: [ProgramName] Сума та валюта: [DepositAmount] [DepositCurrency]
   Строк: [DepositTerm] Спосіб внесення: [PaymentMethod] (якщо обрано картку):
   [CardAmount] [CardCurrency], [CardName] [DateToday] Підпис **\_\_\_** Формат
   запису змінних у тексті можна використовувати будь-який зручний. Наприклад
   {CardName}. Бажаємо успіхів! Чекаємо на посилання на GitHub репозиторій.
